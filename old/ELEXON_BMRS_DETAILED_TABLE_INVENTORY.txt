All tables in the dataset include consistent metadata fields for tracking data provenance and enabling effective analysis:

- **`_dataset`**: Source dataset identifier (e.g., 'BOD', 'FREQ')
- **`_window_from_utc`**: Start timestamp of the data extraction window
- **`_window_to_utc`**: End timestamp of the data extraction window
- **`_ingested_utc`**: Timestamp when the data was ingested into BigQuery
- **`_source_api`**: Source API identifier ('BMRS')
- **`_source_columns`**: Original column names from the source API
- **`_hash_source_cols`**: List of columns used to generate the hash key
- **`_hash_key`**: Unique hash key for deduplication

## DATA ACCESS PATTERNS

### Query Patterns

The most common query patterns for this dataset are:

1. **Time Series Analysis**: Querying specific metrics over time periods
   ```sql
   SELECT settlementDate, settlementPeriod, AVG(frequency) as avg_frequency
   FROM `jibber-jabber-knowledge.uk_energy_insights.bmrs_freq`
   WHERE _window_from_utc BETWEEN '2025-09-01' AND '2025-09-21'
   GROUP BY settlementDate, settlementPeriod
   ORDER BY settlementDate, settlementPeriod
   ```

2. **Market Pricing Analysis**: Analyzing price trends and patterns
   ```sql
   SELECT settlementDate, AVG(systemBuyPrice) as avg_buy, AVG(systemSellPrice) as avg_sell
   FROM `jibber-jabber-knowledge.uk_energy_insights.bmrs_costs`
   WHERE settlementDate BETWEEN '2025-08-01' AND '2025-09-21'
   GROUP BY settlementDate
   ORDER BY settlementDate
   ```

3. **Generator Behavior Analysis**: Examining individual unit behaviors
   ```sql
   SELECT bmUnit, AVG(levelFrom) as avg_level
   FROM `jibber-jabber-knowledge.uk_energy_insights.bmrs_pn`
   WHERE settlementDate = '2025-09-20'
   GROUP BY bmUnit
   ORDER BY avg_level DESC
   LIMIT 20
   ```

4. **Fuel Mix Analysis**: Tracking generation by fuel type
   ```sql
   SELECT settlementDate, fuelType, SUM(generation) as total_gen
   FROM `jibber-jabber-knowledge.uk_energy_insights.bmrs_fuelinst`
   WHERE settlementDate BETWEEN '2025-09-14' AND '2025-09-21'
   GROUP BY settlementDate, fuelType
   ORDER BY settlementDate, total_gen DESC
   ```

## SCHEDULED UPDATES

The dataset is updated via the following schedule:

- **High-Frequency Tables** (FREQ, FUELINST): Every 15 minutes
- **Settlement-Critical Tables** (BOD, BOALF, MILS, MELS): Hourly
- **Daily Forecasts** (NDF, TSDF): Every 6 hours
- **Long-term Forecasts** (UOU2T3YW, UOU2T14D): Daily

All updates are orchestrated through the `ingest_elexon_fixed.py` script with the appropriate time windows.
